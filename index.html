Path function (advanced)

The chain's centre-line as JS: f(t, n_it, n_it_nor, z_stack) → {x, y, z} in mm. t grows continuously along the curve (its scale isn't 1-per-link). n_it is the index (0…count-1) of the link being placed, n_it_nor that index normalized to 0…1 — use either for anything that should step per link, e.g. z: n_it * 3 for a staircase. z only ever lifts a link, it never tilts it (rotation is always worked out in x/y only) — same for z_stack, the height already added by the pin/hole z slider before this link, which is applied automatically on top of your z (it's there to react to, not to return). Changing the Shape selector / Start-radius / Loop-spacing controls above rewrites this function.

Joint anchors (live)

Nudge the pin / hole to retune link pitch & body offset until overlaps (red) clear — the chain re-lays as you type.

Size variation (per part)

JS f(n_it, n_it_nor) → scale. n_it = part index, n_it_nor = that index normalized to 0–1. Return a single number for a uniform scale, or {x, y, z} to scale each axis independently — e.g. stretch a link taller without widening it. Edits apply live; the chain re-spaces so joints still mate. 1 = original size on that axis.

Colour variation (per part)

JS f(n_it, n_it_nor, base) → colour. n_it = part index, n_it_nor = that index normalized to 0–1, base = the part's natural colour {r, g, b} (each 0–1). Return 'invert' to swap this part's colours (e.g. white↔green on a 2-colour model), a colour to tint it (hex 0xff0000, CSS '#ff0000'/'tomato', or {r, g, b}), or base to leave it. The default swaps colours on every 2nd part. A tint multiplies a coloured model's own colours; on a plain part it sets the colour outright. Baked into colour exports (GLB/glTF/OBJ); STL has no colour.
